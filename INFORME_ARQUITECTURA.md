## ?? Patrones de Diseño Implementados

### 1. **Patrón Repository (Repositorio)**

Separa la lógica de acceso a datos de la lógica de negocio.

```csharp
// Interfaz del repositorio
public interface IClienteRepository
{
    Task<int> AddCliente(Cliente client);
    Task<bool> UpdateCliente(Cliente client);
    Task<bool> DeleteCliente(int clienteId);
    Task<List<Cliente>> GetClientes();
    Task<Cliente> GetCliente(int ClientId);
}

// Implementación
public class ClienteRepository : IClienteRepository
{
    private readonly IDataManager<Cliente> _dataManager;
    // Implementación de métodos CRUD
}
```

### 2. **Patrón Service Layer (Capa de Servicios)**

Encapsula la lógica de negocio entre los controladores y los repositorios.

```csharp
// Interfaz del servicio
public interface IClienteService
{
    Task<int> AddCliente(Cliente client);
    Task<bool> UpdateCliente(Cliente client);
    // ...
}

// Implementación
public class ClienteService : IClienteService
{
    private readonly IClienteRepository _clientRepository;

    public ClienteService()
    {
        this._clientRepository = new ClienteRepository();
    }
}
```

### 3. **Patrón Generic Data Manager**

Implementación genérica para manejo de datos con reflexión.

```csharp
public class MySqlDataManager<T> : IDataManager<T> where T : class, new()
{
    public async Task<List<T>> ExecReader(string spName, Dictionary<string, object> parameters);
    public async Task<int> ExecNonQuery(string spName, Dictionary<string, object> parameters);
}
```

### 4. **Patrón Template Method (BaseController)**

Clase base que proporciona funcionalidad común para todos los controladores.

```csharp
public class BaseController : System.Web.UI.Page
```

## ??? Base de Datos

### Motor de Base de Datos

- **Sistema**: MySQL
- **Versión del Driver**: MySql.Data v6.5.4.0

### Implementación del Data Manager

```csharp
public class MySqlDataManager<T> : IDataManager<T> where T : class, new()
{
    // Usa reflexión para mapear resultados a objetos
    private List<T> ConvertToObject(IDataReader reader)
    {
        var results = new List<T>();
        var properties = typeof(T).GetProperties();

        while (reader.Read())
        {
            var item = Activator.CreateInstance<T>();
            foreach (var property in typeof(T).GetProperties())
            {
                // Mapeo automático por nombre de propiedad
                property.SetValue(item, reader[property.Name], null);
            }
            results.Add(item);
        }
        return results;
    }
}
```

## ?? Seguridad

### Autenticación

- **Sistema de tokens personalizados** basado en GUID + timestamps
- **Encriptación AES** para tokens
- **Hash de contraseñas** con HMACSHA512

### Flujo de Autenticación

```
1. Usuario envía credenciales
2. UsuarioService.Authenticate() verifica hash
3. Se genera token: UserGuid + Timestamp + Expiración
4. Token encriptado con AES
5. Cada request valida token en BaseController.VerifySecureToken()
```

### Implementación de Token

```csharp
// Generación de token
DateTime now = DateTime.Now;
long expire = now.AddHours(12).Ticks;
string tokenData = user.UserGuid + ";" + now.Ticks.ToString() + ";" + expire.ToString();
token = Cryptography.Encrypt(tokenData, cryptPassword);

// Validación
public static bool VerifySecureToken()
{
    string token = headers["Authorization"];
    bool validToken = _UsuarioService.VerifyToken(token).GetAwaiter().GetResult();
    return validToken;
}
```

## ?? Observaciones y Recomendaciones

### Áreas de Mejora

1. **Inyección de Dependencias**
   - Actualmente las dependencias se crean manualmente (`new Repository()`)
   - Se recomienda implementar un contenedor DI (Unity, Autofac, o migrar a .NET Core)

2. **Manejo de Conexión a BD**
   - La cadena de conexión está hardcodeada
   - Se recomienda usar `Web.config` o `appsettings.json`

3. **Seguridad**
   - El token personalizado podría reemplazarse por JWT estándar
   - Considerar HTTPS obligatorio

4. **Manejo de Excepciones**
   - Hay bloques catch vacíos o que solo hacen `throw`
   - Implementar logging estructurado (Serilog, NLog)

5. **Async/Await**
   - Se usa `Task.Start()` + `Wait()` en lugar de `await`
   - Puede causar deadlocks en contextos de sincronización

6. **Versión del Framework**
   - .NET Framework 4.0 es muy antiguo
   - Considerar migración a .NET 6/8

### Fortalezas

? Separación clara de responsabilidades (capas)  
? Uso de interfaces para abstracción  
? Patrón Repository bien implementado  
? Modelo de respuesta estandarizado  
? Frontend moderno con Angular  
? Seguridad por tokens implementada

## ?? Contexto del Negocio

La aplicación **EPSEPAR** es un sistema de gestión para una **EPS (Entidad Promotora de Salud)** que permite:

- Gestión de **clientes** (empresas afiliadas)
- Administración de **empleados** y sus **beneficiarios**
- Control de **planillas** de pago
- Manejo de **ARLs, AFPs, EPSs y Cajas de Compensación**
- Gestión de **sedes**
- Estadísticas y reportes (Dashboard)
- Control de **pagos** y **mora** de clientes
